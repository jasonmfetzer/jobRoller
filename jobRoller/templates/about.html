Goals

Our goals for this project were simple. As IIT continues to add courses, we wanted to be able to reverse engineer a plan of study based on a Job Description that met CS requirements at IIT. We came up with a simple user story where a user could enter URL from Built In Chicago, and we would scrape the site to query the database and see if there were relevant courses to that technology. For example, if a site listed AWS, we wanted to display "Cloud Computing."


As the project grew, we realized that limiting to that simple functionality already required a database, so we added basic search functionality for keyword. We implemented a Keyword Search and Basic Course Search on our site. We followed a MVC pattern architecture where the models were courses and keywords, views were the controller functions, and the template system actually serves as the UI with a nice template tagging system.

Successes

We suceeded in developing our application versus non-functional. We were able to correctly query database results with filters, and we were able to actually pull back the key technologies from a job link and render them to the page.


Testing

We have minimalist checks in the application, mainly due to time constraints, but we verify the HTTPResponse codes, and have used some conditionals in our functions. 404 Errors will simply render a 'Page Not Found' warning in production. Unit Tests are a work in progress, as well as implementing TestRunners or Selenium in the long run.


Reusability

We leveraged the Django Framework in this application, and while it is not completely tested, one nice feature of the framework is the reusability of our components. For instance, in our app, the 'soupify' search, while it is part of our jobView app as a whole, can be reused in other projects. This is a nice feature, say if we were building something for other area universities to tap into.


Failures

We failed in the fact of separation or abstraction. We were not able to render the database objects based on keywords. A user would have to get the keywords and then go to the /kw_search and enter the keyword to see if there are relevant courses. Functionality is there, but the Mixin did not work properly. Additionally, it was our goal to be able to actually add courses to a dynamic list, so that users could save the data. This functionality is a work in progress.
